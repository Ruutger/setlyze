setlyze.config — Configuration manager¶

Author:Serrano Pereira, Adam van Adrichem, Fedde Schaeffer
Release:1.0
Date:April 05, 2013

Module Contents¶

This modules provides application-wide access to SETLyze’s configuration variables.

This module provides an object cfg for handling a fixed set of configuration variables for SETLyze. The big advantage is that this makes the variables available across all modules. Here is a small usage example,

>>> import setlyze.config
>>> setlyze.config.cfg.set('alpha-level', 0.01)
>>> print "The alpha level for statistical tests is set to %s" % setlyze.config.cfg.get('alpha-level')
The alpha level for statistical tests is set to 0.01

Importing this module in a different module gives access to the same cfg object, and thus its variables can be obtained or manipulated using its ConfigManager.get() and ConfigManager.set() methods.

class setlyze.config.ConfigManager[source]¶

Class for managing SETLyze’s configuration variables.

An instance of this class provides access to a fixed set of variables that need to be accessable across SETLyze’s modules. By importing this module, one instance of this class is created. Subsequent imports in other modules provides access to that same instance.

The method set() is used to change the value of variables. The method get() is used to get the value of a variable.

All configurations and their default values can be found in global variable DEFAULT_CONFIG of this module.

Design Part: 1.57

get(key)[source]¶

Return the value for the configuration with name key.

read_from_file()[source]¶

Try to read settings from a configuration file.

The default location of the configuration file is ~/.setlyze/setlyze.cfg.

save_to_file()[source]¶

Save user customizable settings to a configuration file.

The configuration file is by default saved to ~/.setlyze/setlyze.cfg.

set(key, value)[source]¶

Set the configuration with name key to value.

set_data_source(source)[source]¶

Set the configuration with name data-source to source.

Possible values for source are setl-database and data-files. The value of this configuration tells the application where to look for SETL data. This is especially used by the database module.

If an unknown data source is given, an error is printed.
